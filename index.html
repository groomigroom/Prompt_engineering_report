프롬프트 엔지니어링

▪ 대규모 언어 모델(LLM)이나 이미지 생성 AI 같은 생성형 AI가 사용자가 원하는 가장 정확하고 고품질의 결과물을 내놓도록 '최적의 입력값(프롬프트)'을 설계하고 테스트하는 기술과 과정
▪ AI에게 단순히 "OO에 대해 써줘"라고 지시하는 대신, 역할 부여, 맥락 제공, 구체적인 형식 지정 등 섬세한 지침을 주어 AI의 결과물을 극대화하는 소통의 기술입니다.
▪ 프롬프트(Prompt): AI에게 내리는 질문이나 명령어
▪ 엔지니어링(Engineering): 시행착오를 거쳐 AI의 특성(장단점)을 이해하고, 원하는 결과가 나오도록 체계적으로 프롬프트를 만드는 작업

프롬프트 작성의 주요 기법 (프롬프트 엔지니어링)

▪ 명확한 역할 부여: "너는 10년 차 마케터야"와 같이 페르소나를 지정합니다.
▪ 배경과 맥락 제공: 필요한 문서나 조건, 대상 독자 등을 상세히 입력합니다.
▪ 출력 형식 및 길이 제한: "핵심 내용을 표로 작성해 줘" 또는 "500자 이내로 요약해 줘"처럼 형태를 규정합니다.
▪ 단계별 지시 (생각할 시간 주기): 복잡한 문제는 단계별로 나누어 차근차근 해결하도록 지시하여 논리적 오류를 줄입니다.
▪ 예시 제공(Few-Shot): 원하는 결과물과 비슷한 형식의 예시를 프롬프트에 직접 포함하여 AI가 패턴을 파악하게 합니다.

AI에이전트 엔지니어링

▪ AI 에이전트 엔지니어링은 사용자가 설정한 목표를 달성하기 위해 인공지능이 스스로 계획을 세우고, 외부 도구와 데이터를 활용하며, 의사결정을 내리는 자율형 소프트웨어 시스템을 구축하는 기술입니다.
▪ 이는 단순한 프롬프트 조합을 넘어 안정적이고 독립적인 AI 애플리케이션을 설계하고 운영하는 과정 전체를 포괄합니다.
▪ 핵심 구성 요소 및 작동 방식추론 및 계획 (Reasoning & Planning): 대형언어모델(LLM)을 통해 주어진 목표를 분석하고, 이를 달성하기 위한 하위 작업들의 순서와 전략을 스스로 수립합니다.
▪ 도구 연동 (Tool Use): 웹 검색, 데이터베이스 조회, 사내 API 호출 등 AI가 직접 수행할 수 없는 작업들을 외부 도구와 연결하여 해결합니다.
▪ 메모리 (Memory): 이전 대화의 문맥(Context)을 기억하거나 작업 수행 과정에서 발생하는 중간 결과를 저장하여 연속성 있는 작업을 가능하게 합니다.
▪ 오케스트레이션 및 멀티 에이전트: 단일 AI의 한계를 극복하기 위해 여러 에이전트가 각각의 역할(예: 기획, 코딩, 테스트)을 나누어 협업하도록 아키텍처를 구성합니다.
▪ 전통적인 개발과의 차이점은 기존 소프트웨어가 정해진 규칙(Rule)대로만 동작했다면, AI 에이전트 시스템은 비결정적(Non-deterministic)이고 확률적으로 작동합니다. 가드레일 및 보안 (Governance): AI가 할루시네이션(환각)을 일으키거나 의도하지 않은 위험한 행동을 하지 않도록 통제선을 설계합니다. 환경 모니터링 (Monitoring): 기존의 시스템 서버 점검을 넘어, AI가 올바른 도구를 선택했는지, 논리적 오류는 없는지 의미론적으로 평가하고 피드백 루프를 구축합니다. AI 에이전트 엔지니어링은 아이디어를 프로덕션 수준의 신뢰할 수 있는 시스템으로 전환하여, 단순한 챗봇이 아닌 스스로 일하는 '지능형 동료'를 만들어내는 핵심 기술입니다.

AI에이전트 엔지니어링

▪ 하네스 엔지니어링(Harness Engineering)은 AI 에이전트가 사람의 직접적인 감독 없이도 자율적으로 복잡한 업무를 안전하고 일관되게 수행할 수 있도록, AI를 둘러싼 외부 실행 환경, 제약 조건, 피드백 루프 전체를 설계하는 방법론입니다. 과거의 '프롬프트 엔지니어링'이 AI에게 "무엇을 어떻게 말할까(질문법)"를 다뤘다면, 하네스 엔지니어링은 AI가 일할 수 있는 "업무 환경과 규칙(시스템)"을 설계하는 데 초점을 맞춥니다. '하네스(마구)'가 말의 힘을 제어해 올바른 방향으로 이끌듯, AI의 능력이 엉뚱한 곳으로 새지 않고 목적지에 도달하도록 안전망을 구축하는 것이 핵심입니다. 하네스를 잘 구축하면 AI는 정해진 규약 안에서만 행동하고 스스로 실수를 수정하며 장기 프로젝트를 완수할 수 있습니다. 하네스 엔지니어링의 핵심 구성 요소 컨텍스트 및 메모리 설계: AI가 작업을 수행할 때 참고할 수 있도록 가이드라인(CLAUDE.md, AGENTS.md)과 이전의 실수 기록(Bug Log)을 짧고 구조화된 문서로 제공합니다. 도구(Tool) 연결: MCP(Model Context Protocol) 등을 활용해 웹 검색, 캘린더, 파일 시스템, 데이터베이스 등 AI가 필요한 외부 도구를 알맞게 연결합니다. 아키텍처 제약 (가드레일): 린터, 디렉토리 규칙 등을 통해 AI가 구조적으로 잘못된 행동을 하거나 위험한 작업을 시도하는 것을 원천 차단합니다. 피드백 루프 (검증 시스템): AI가 작업을 완료했다고 선언하기 전, 요구사항에 맞게 작성되었는지 자동 테스트(CI)와 검증(Hook) 과정을 거치도록 강제합니다. 적용 사례 및 전망개발 분야에서는 코드를 직접 작성하는 엔지니어의 역할이 줄어들고, AI가 코드를 잘 짤 수 있도록 위와 같은 작업 환경과 워크플로우를 설계하는 '하네스 엔지니어'의 역할이 점차 중요해지고 있습니다. 단발성 질문이 아닌 복잡한 장기 프로젝트를 AI 에이전트에게 맡길 때 필수적인 역량으로 꼽힙니다.
